Embora os nomes sejam parecidos, **Git** e **GitHub** não são a mesma coisa.

| Git | GitHub |
|------|---------|
| É um sistema de controle de versão. | É uma plataforma de hospedagem de repositórios Git. |
| Funciona localmente no computador. | Funciona na nuvem (online). |
| Controla o histórico de alterações dos arquivos. | Permite compartilhar projetos e colaborar com outras pessoas. |
| Gerencia branches, commits e versões. | Oferece recursos como Pull Requests, Issues, GitHub Actions e Wiki. |
| Pode ser utilizado sem conexão com a internet. | Geralmente requer acesso à internet para sincronizar alterações. |

## Git

O **Git** é a ferramenta responsável por controlar as versões de um projeto. Com ele, é possível registrar alterações, criar branches, desfazer modificações e acompanhar todo o histórico de desenvolvimento.

Exemplo de comandos do Git:

```bash
git init
git add .
git commit -m "Primeiro commit"
```

## GitHub

O **GitHub** é uma plataforma online que hospeda repositórios Git. Ele permite que desenvolvedores compartilhem seus projetos, trabalhem em equipe e utilizem recursos de colaboração, como Pull Requests e revisão de código.

Exemplo de uso do GitHub:

- Hospedar um projeto.
- Compartilhar o repositório com outros desenvolvedores.
- Abrir um Pull Request para revisar alterações.
- Gerenciar tarefas por meio de Issues.

## Exemplo prático

Imagine que você está escrevendo um trabalho da faculdade.

- O **Git** funciona como o histórico de versões do documento, registrando todas as alterações feitas.
- O **GitHub** funciona como um espaço na internet onde esse trabalho é armazenado, compartilhado e editado em conjunto com outras pessoas.

## Resumo

> **Git** é a ferramenta que controla as versões de um projeto.
> **GitHub** é a plataforma que hospeda repositórios Git e facilita a colaboração entre desenvolvedores.
